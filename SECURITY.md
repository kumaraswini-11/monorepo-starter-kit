# Security Policy

We take the security of monorepo-starter-kit seriously. Thank you for helping
keep it and its users safe through responsible, coordinated disclosure.

## Supported Versions

Security fixes are applied to the most recent release (and the `main` branch).
Until the project reaches a stable `1.0.0`, only the latest version is
supported — please make sure you are on it before reporting.

| Version                 | Supported          |
| ----------------------- | ------------------ |
| Latest release / `main` | :white_check_mark: |
| Any older version       | :x:                |

A formal version-support policy will be published once the project reaches
`1.0.0`.

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.** Disclosing a vulnerability publicly before a
fix is available puts users at risk.

Instead, report it privately through **coordinated disclosure**, using either:

1. **GitHub Private Vulnerability Reporting** (preferred) — open the
   repository's **Security** tab and choose **Report a vulnerability**. (If the
   option is not visible, it can be enabled in the repository's security
   settings; otherwise use email below.)
2. **Email** — <kumaraswini.11.11@gmail.com>

### What to include

To help us triage and resolve the issue quickly, please include as much of the
following as you can:

- The type of issue (e.g. XSS, SQL injection, authentication bypass, RCE, SSRF)
- The affected version(s), and the location in the source (tag/branch/commit, or
  a direct URL)
- Full path(s) of the affected source file(s)
- The impact — how an attacker might exploit it, and what they could achieve
- Step-by-step instructions to reproduce the issue
- Any special configuration required to reproduce it
- Relevant logs, screenshots, or proof-of-concept / exploit code (if available)

## Response & Disclosure Process

- **Acknowledgment:** we will confirm receipt of your report within **72 hours**.
- **Assessment:** we will investigate, determine severity, and keep you updated
  on our progress.
- **Fix & disclosure:** we work toward a fix and a **coordinated disclosure**,
  typically within **90 days** of the report, and will agree on public
  disclosure timing with you.
- **Credit:** with your consent, we are glad to publicly credit you for the
  discovery once a fix is released.

Thank you for helping keep monorepo-starter-kit and its users safe.
